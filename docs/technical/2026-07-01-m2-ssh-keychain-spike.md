# M2 SSH And Keychain Technical Spike

## SSH Adapter Choice

M2 uses the system OpenSSH client from the local macOS environment. The service launches `ssh` with `child_process.spawn()` and an argument array. It does not use a local shell.

Reasons:

- It preserves user-managed `~/.ssh/config`, identities, agents, and `known_hosts` behavior.
- It avoids storing private keys or passphrases in Env Config Lens.
- It keeps the M2 implementation small and auditable for a local-first desktop tool.

## Host Verification

The adapter explicitly passes:

- `StrictHostKeyChecking=yes`
- `PasswordAuthentication=no`
- `KbdInteractiveAuthentication=no`
- `PreferredAuthentications=publickey`

It never passes `StrictHostKeyChecking=no`. Unknown or changed host keys fail before any env content is returned.

## Remote Read Safety

The UI stores one configured `remoteEnvPath`. The backend does not expose arbitrary remote command input.

The remote read command is generated by the backend as:

```text
cat -- '<remoteEnvPath>'
```

The path is single-quoted for a POSIX remote shell and rejects NUL or newline characters. The local process is still launched without a shell, so local command injection is not possible.

Remote stdout is captured in memory for the current request only. It is not written to a temp file or normal local file path.

## Keychain Reference

SQLite stores only an optional Keychain reference:

- service name
- account name

The passphrase itself is retrieved by a checked-in `SSH_ASKPASS` helper that calls macOS `security find-generic-password -w`. The helper receives only the service/account reference through environment variables and prints the passphrase directly to OpenSSH when OpenSSH asks for a private-key passphrase.

The app does not store passphrases, private key contents, env contents, or parsed env values in SQLite.

## Failure Categories

SSH failures are normalized before API responses:

- `connection_failed`
- `auth_failed`
- `permission_denied`
- `path_not_found`
- `parse_failed`
- `unknown_error`

Sanitized responses do not echo remote stdout, env file contents, private key material, or passphrases.
